PRODUCT REQUIREMENTS DOCUMENT (PRD)
EduManage — Course Management System
Version: 2.0 | Date: March 2026 | Status: Delivered
========================================

1. PRODUCT OVERVIEW
-------------------
EduManage is a web-based Course Management System (CMS) designed to manage
academic courses, instructors, and student enrollments within an educational
institution.

Technology Stack:
  Frontend : HTML, Tailwind CSS, JavaScript
  Backend  : PHP (PDO)
  Database : MySQL
  Server   : Apache (XAMPP)
  Icons    : Google Material Symbols
  Fonts    : Google Fonts (Inter)


2. PROBLEM STATEMENT
--------------------
Many educational institutions rely on manual processes (spreadsheets,
paperwork) to manage courses and student records. This creates:
  - Difficulty managing large numbers of students and courses
  - Data redundancy and human errors
  - Inefficient communication between instructors and students
  - Lack of centralized access to course and enrollment information

EduManage solves all of these with a structured, secure, role-based platform.


3. OBJECTIVES
-------------
  - Provide a centralized system for managing courses, students, instructors
  - Allow students to browse and enroll in courses online
  - Enable instructors to manage their courses and view enrolled students
  - Allow administrators to control the entire platform
  - Maintain secure session-based authentication for all users


4. TARGET USERS & ROLES
------------------------
4.1 Administrator
  - Manage users (add, edit, delete)
  - Create, edit, and delete courses
  - Assign instructors to courses
  - Monitor enrollments and system activity

4.2 Instructor
  - View assigned courses with enrollment counts
  - View list of enrolled students per course
  - Update course details and descriptions

4.3 Student
  - Register and log in
  - Browse the course catalog with images and filters
  - Enroll in available courses
  - View enrolled and recommended courses on dashboard


5. SYSTEM SCOPE
---------------
In Scope:
  - User authentication (register/login/logout) with role-based access
  - Admin dashboard with live stats and enrollment trend chart
  - Full course CRUD with image upload
  - Student enrollment system with duplicate prevention
  - Instructor portal (assigned courses, student lists)
  - Student portal (catalog, my courses, personalized dashboard)
  - Feedback/contact system
  - Dark/light mode toggle
  - Responsive design for all screen sizes

Out of Scope:
  - Video streaming or LMS content delivery
  - Payment gateway integration
  - AI-based course recommendations
  - Mobile native applications


6. FUNCTIONAL REQUIREMENTS
---------------------------
6.1 User Registration & Login
  - Registration: name, email, password, role
  - Password hashing with password_hash() (bcrypt)
  - Email uniqueness validation
  - Session-based login with role redirection
  - Logout destroys session

6.2 Admin Dashboard
  - Stat cards: Total Users, Total Courses, Enrollments, Students
  - Enrollment Trends bar chart (last 7 days) with visible counts
  - System Activity feed (recent enrollments)
  - Recent Enrollments table

6.3 Course Management
  - Admin and instructors can create/edit/delete courses
  - Fields: title, description, category, academic year, status, instructor, image
  - Course image upload (JPG/PNG/WEBP, max 5MB)
  - Filtering by category, year, status

6.4 Student Enrollment
  - Browse course catalog with images and metadata
  - One-click enrollment with duplicate prevention
  - Enrolled courses on dashboard and "My Courses" page
  - Recommended and new courses on student dashboard

6.5 Instructor Dashboard
  - Stats: Total Students, Active Courses, Avg Completion, Rating
  - Grid of assigned courses with images and student counts
  - Recent students table per course

6.6 Student Dashboard
  - Enrolled courses grid with images and year badges
  - Recommended courses list with images
  - New arrivals scroll section
  - Real-time search filter

6.7 Public Pages
  - Home: Hero with dynamic course info + featured courses grid
  - Courses: Full listing with search and filters
  - About, Contact, Help Center, Privacy Policy


7. NON-FUNCTIONAL REQUIREMENTS
--------------------------------
Performance:
  - Page load time under 3 seconds
  - Efficient SQL queries using PDO prepared statements

Security:
  - password_hash() / password_verify() for credentials
  - PDO prepared statements (SQL injection prevention)
  - htmlspecialchars() on all output (XSS prevention)
  - Session verification on every protected page
  - File upload: type whitelist + max size (5MB)

Usability:
  - Clean modern UI with Tailwind CSS
  - Dark/light mode support
  - Responsive (mobile to desktop)

Scalability:
  - Modular PHP file structure
  - Centralized DB connection via config/db.php


8. DATABASE DESIGN
------------------
Table: users
  id          INT (PK, AUTO_INCREMENT)
  name        VARCHAR(100)
  email       VARCHAR(150) UNIQUE
  password    VARCHAR(255)
  role        ENUM('admin','instructor','student')
  created_at  DATETIME

Table: courses
  id             INT (PK, AUTO_INCREMENT)
  title          VARCHAR(200)
  description    TEXT
  category       VARCHAR(100)
  academic_year  VARCHAR(50)
  status         ENUM('active','inactive','draft')
  instructor_id  INT (FK -> users.id)
  image          VARCHAR(255)
  created_at     DATETIME

Table: enrollments
  id           INT (PK, AUTO_INCREMENT)
  student_id   INT (FK -> users.id)
  course_id    INT (FK -> courses.id)
  enrolled_at  DATETIME

Table: customer_feedback
  id            INT (PK)
  visitor_name  VARCHAR(100)
  email         VARCHAR(150)
  category      VARCHAR(100)
  comments      TEXT
  created_at    DATETIME


9. SYSTEM ARCHITECTURE
-----------------------
Three-Tier Architecture:

  [Browser / Tailwind CSS / JavaScript]
              |
              | HTTP
              v
  [Apache -> PHP Application Layer]
              |
              | PDO
              v
  [MySQL Database Layer]

Folder Structure:
  CMS/
  |-- admin/           Admin panel pages
  |-- api/             API handlers (course, enrollment actions)
  |-- assets/          CSS, JS, images (logo)
  |-- auth/            Login & Register
  |-- config/          db.php, session.php
  |-- includes/        Shared sidebar per role
  |-- instructor/      Instructor panel pages
  |-- student/         Student panel pages
  |-- uploads/         Course image uploads
  |-- index.php        Public landing page
  |-- courses.php      Public course listing
  |-- about.php
  |-- contact.php
  |-- help_center.php
  |-- privacy.php
  |-- PRD.md
  |-- PRD.txt
  |-- README.md


10. USER FLOWS
--------------
Student Flow:
  1. Register -> redirected to student dashboard
  2. Browse course catalog -> filter/search
  3. Click Enroll -> confirmed
  4. View enrolled courses on My Courses page

Instructor Flow:
  1. Login -> instructor dashboard
  2. View assigned courses + enrollment stats
  3. View student list per course

Admin Flow:
  1. Login -> admin dashboard (stats + chart + activity)
  2. Manage Courses -> add/edit/delete + image upload
  3. Manage Users -> assign roles
  4. Monitor Enrollments, Feedback, System Logs

12. SECURITY MEASURES
----------------------
  - Passwords hashed with password_hash() (bcrypt)
  - All SQL uses PDO prepared statements
  - All output escaped with htmlspecialchars()
  - Sessions verified on every protected page via requireRole()
  - File uploads: type-checked (jpg, png, webp) + 5MB limit
  - Input sanitized via custom sanitize() helper
